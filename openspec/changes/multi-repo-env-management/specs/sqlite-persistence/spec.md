## ADDED Requirements

### 需求:SQLite 数据库初始化
系统启动时必须初始化 SQLite 数据库，自动创建所需的表结构。数据库文件必须存储在 `data/review-env.db`。

#### 场景:首次启动
- **当** 系统首次启动且数据库文件不存在
- **那么** 系统必须创建数据库文件并执行建表语句，创建 projects、project_images、containers 三张表

#### 场景:数据库已存在
- **当** 系统启动且数据库文件已存在
- **那么** 系统必须直接使用现有数据库，不重复建表

### 需求:容器状态持久化
容器的创建和销毁必须同步更新数据库记录。系统禁止仅依赖内存状态管理容器。

#### 场景:创建容器时写入数据库
- **当** 成功创建 Docker 容器
- **那么** 系统必须在 containers 表中插入记录，包含 project_id、mr_iid、branch、image_id、container_id、ports

#### 场景:销毁容器时删除记录
- **当** 容器被停止并移除
- **那么** 系统必须从 containers 表中删除对应记录

#### 场景:服务重启后恢复
- **当** 服务重启
- **那么** 系统必须从 containers 表读取记录，并与 Docker 实际状态校验，删除 Docker 中已不存在的容器记录

### 需求:数据库唯一性约束
数据库必须通过唯一约束确保数据一致性。

#### 场景:同一项目同一 MR 唯一容器
- **当** 尝试为项目 42 的 MR 17 创建第二个容器记录
- **那么** 数据库必须拒绝插入（UNIQUE 约束 project_id + mr_iid）

#### 场景:同一项目镜像名唯一
- **当** 尝试为项目 42 添加第二个 name="claude-code" 的镜像
- **那么** 数据库必须拒绝插入（UNIQUE 约束 project_id + name）

#### 场景:gitlab_project_id 全局唯一
- **当** 尝试添加 gitlab_project_id 已存在的项目
- **那么** 数据库必须拒绝插入（UNIQUE 约束）
