## ADDED Requirements

### 需求:SPA 管理界面框架
系统必须提供基于 React + shadcn 的 SPA 管理界面，通过 `/admin` 路径访问。Express 必须托管 SPA 的静态文件并支持客户端路由 fallback。

#### 场景:访问管理界面
- **当** 用户访问 `/admin` 或 `/admin/*`
- **那么** 系统必须返回 SPA 的 index.html，由前端路由处理页面渲染

#### 场景:SPA 静态资源
- **当** 浏览器请求 `/admin/assets/*`
- **那么** 系统必须返回对应的 JS/CSS/图片等静态文件

### 需求:项目管理页面
管理界面必须提供项目的增删改查功能。

#### 场景:项目列表页
- **当** 用户访问 `/admin/projects`
- **那么** 页面必须展示所有已配置项目的列表，包含项目名称、GitLab 项目路径、镜像数量、容器数量

#### 场景:添加项目
- **当** 用户在项目管理页面点击"添加项目"并填写表单
- **那么** 页面必须调用 `POST /api/projects` 创建项目，成功后刷新列表

#### 场景:编辑项目
- **当** 用户点击某个项目的编辑按钮
- **那么** 页面必须展示项目配置表单，允许修改名称、PAT、Webhook Secret 等字段

#### 场景:删除项目
- **当** 用户点击删除并确认
- **那么** 页面必须调用 `DELETE /api/projects/:id`，成功后从列表中移除

### 需求:镜像配置页面
管理界面必须提供项目镜像的配置功能，作为项目详情的一部分。

#### 场景:查看项目镜像
- **当** 用户进入项目详情页
- **那么** 页面必须展示该项目已配置的所有镜像列表

#### 场景:添加镜像
- **当** 用户点击"添加镜像"并填写表单（name、display_name、image、env_vars）
- **那么** 页面必须调用 `POST /api/projects/:id/images` 创建镜像配置

#### 场景:编辑镜像
- **当** 用户编辑镜像配置
- **那么** 页面必须调用 `PUT /api/projects/:id/images/:imageId` 更新配置

#### 场景:启用/禁用镜像
- **当** 用户切换镜像的启用状态
- **那么** 页面必须调用 API 更新 enabled 字段

### 需求:容器监控页面
管理界面必须提供运行中容器的查看和管理功能。

#### 场景:容器列表
- **当** 用户访问 `/admin/containers`
- **那么** 页面必须展示所有运行中的容器，包含项目名称、MR 编号、使用的镜像、运行时长

#### 场景:手动停止容器
- **当** 用户点击某个容器的停止按钮并确认
- **那么** 页面必须调用 `DELETE /api/containers/:id` 停止并移除容器

#### 场景:全局统计
- **当** 用户访问管理界面首页
- **那么** 页面必须展示全局统计信息：活跃容器数、已配置项目数、最大容器限制
