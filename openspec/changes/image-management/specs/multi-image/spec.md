## MODIFIED Requirements

### 需求:项目镜像配置
每个项目必须支持配置多个 Docker 镜像，每个镜像代表一种 AI 工具环境。镜像配置必须包含名称、显示名、Docker 镜像地址、额外环境变量和排序顺序。

#### 场景:添加镜像配置
- **当** 管理员通过 `POST /api/projects/:id/images` 提交镜像配置（name、display_name、image、env_vars）
- **那么** 系统必须将镜像配置保存到数据库并关联到对应项目

#### 场景:镜像名称唯一性
- **当** 同一项目下添加重复 name 的镜像
- **那么** 系统必须返回 409 冲突错误

#### 场景:获取项目镜像列表
- **当** 请求 `GET /api/projects/:id/images`
- **那么** 系统必须返回该项目所有已启用的镜像配置，按 sort_order 排序

#### 场景:禁用镜像
- **当** 管理员将某个镜像的 enabled 设为 false
- **那么** 该镜像禁止出现在终端页面的工具选择列表中

#### 场景:添加镜像表单 - 镜像选择
- **当** 用户在项目详情页点击"添加镜像"
- **那么** Docker 镜像字段必须展示为下拉选择组件，数据源为 `GET /api/docker/images` 返回的宿主机镜像列表，支持搜索过滤

#### 场景:添加镜像表单 - 环境变量编辑
- **当** 用户在添加镜像表单中配置环境变量
- **那么** 页面必须展示 key-value 表格组件，每行包含变量名输入框、值输入框和删除按钮，底部提供"添加变量"按钮。提交时前端必须将表格数据序列化为 JSON 字符串
