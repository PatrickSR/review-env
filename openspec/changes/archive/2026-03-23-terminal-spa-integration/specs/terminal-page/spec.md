## ADDED Requirements

### 需求:终端页面 React 组件
系统必须提供 `/mr/:projectId/:mrIid` 路由对应的 React 页面组件，替代原有的独立 HTML 页面。该组件必须纳入 SPA 的 Layout 中，共享侧栏导航。

#### 场景:访问终端页面
- **当** 用户访问 `/mr/42/17`
- **那么** SPA 必须渲染终端页面组件，左侧显示侧栏导航，中间为终端主内容区

### 需求:MR 状态验证
终端页面加载时必须先验证 MR 的有效性。如果 MR 不存在、已关闭或已合并，必须显示友好的错误提示。

#### 场景:MR 有效（open 状态）
- **当** 终端页面加载且 MR 验证返回 valid=true
- **那么** 页面继续正常流程（查询容器状态或显示镜像选择）

#### 场景:MR 无效（已关闭/已合并/不存在）
- **当** 终端页面加载且 MR 验证返回 valid=false
- **那么** 页面必须在中间区域显示错误提示："该 MR 已关闭或不存在，请检查 MR 状态"
- **并且** 禁止显示镜像选择或终端 iframe

### 需求:镜像选择流程
当 MR 有效但无运行中的容器时，终端页面必须显示镜像选择界面，用户选择后启动容器。

#### 场景:无容器，显示镜像选择
- **当** MR 有效且容器状态为 not_found
- **那么** 页面必须在中间区域显示该项目可用的镜像列表（卡片形式）
- **并且** 用户点击镜像卡片后调用 `POST /mr/:projectId/:mrIid/start` 启动容器

#### 场景:无可用镜像
- **当** MR 有效但项目未配置任何启用的镜像
- **那么** 页面必须显示提示："该项目尚未配置任何镜像，请联系管理员"

### 需求:容器状态轮询与终端显示
当容器存在时，终端页面必须轮询容器状态，就绪后通过 iframe 嵌入 ttyd 终端。

#### 场景:容器就绪
- **当** 容器状态轮询返回 status=ready 且 ports 包含 7681 映射
- **那么** 页面必须渲染 iframe，src 设置为 `http://${location.hostname}:${ports[7681]}/`
- **并且** iframe 必须占满中间区域的全部可用空间

#### 场景:容器初始化中
- **当** 容器状态为 creating 或 initializing
- **那么** 页面必须显示加载动画和状态文字（"正在创建容器..."/"正在初始化环境..."）
- **并且** 每 2 秒轮询一次状态

#### 场景:容器出错
- **当** 容器状态为 error
- **那么** 页面必须显示错误信息

### 需求:右侧可收起信息面板
终端页面必须包含右侧信息面板，默认收起，hover 或点击时展开。

#### 场景:面板收起状态
- **当** 终端页面加载完成
- **那么** 右侧面板必须默认收起，显示为一个窄条（含展开图标）
- **并且** 终端 iframe 占据除侧栏外的全部宽度

#### 场景:面板展开
- **当** 用户 hover 或点击右侧窄条
- **那么** 面板必须展开，显示端口映射、项目信息、MR 信息
- **并且** 展开/收起必须有平滑的过渡动画

#### 场景:端口映射显示
- **当** 面板展开且容器有端口映射
- **那么** 面板必须以列表形式显示所有端口映射（容器端口 → 宿主机端口）
- **并且** 每个端口映射必须可点击，点击后在新标签页打开 `http://${hostname}:${hostPort}`

#### 场景:操作按钮
- **当** 面板展开且容器正在运行
- **那么** 面板必须显示"停止容器"和"切换镜像"按钮
- **并且** 停止容器必须弹出确认对话框
