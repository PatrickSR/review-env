## MODIFIED Requirements

### 需求:项目镜像配置
每个项目必须支持配置多个 Docker 镜像，每个镜像代表一种 AI 工具环境。镜像配置必须包含名称、显示名、Docker 镜像地址、额外环境变量、映射端口和排序顺序。

#### 场景:添加镜像配置
- **当** 管理员通过 `POST /api/projects/:id/images` 提交镜像配置（name、display_name、image、env_vars、ports）
- **那么** 系统必须将镜像配置保存到数据库并关联到对应项目

#### 场景:镜像名称唯一性
- **当** 同一项目下添加重复 name 的镜像
- **那么** 系统必须返回 409 冲突错误

#### 场景:获取项目镜像列表
- **当** 请求 `GET /api/projects/:id/images`
- **那么** 系统必须返回该项目所有已启用的镜像配置（包含 ports 字段），按 sort_order 排序

#### 场景:禁用镜像
- **当** 管理员将某个镜像的 enabled 设为 false
- **那么** 该镜像禁止出现在终端页面的工具选择列表中

#### 场景:编辑镜像配置
- **当** 管理员通过 `PUT /api/projects/:id/images/:imageId` 提交更新字段
- **那么** 系统必须更新对应镜像的配置并返回更新后的完整记录

#### 场景:添加镜像表单 - 使用 Dialog
- **当** 用户在项目详情页点击"添加镜像"
- **那么** 系统必须弹出 Dialog（Modal）表单，包含标识名、显示名、Docker 镜像选择、映射端口和环境变量字段

#### 场景:编辑镜像表单 - 使用 Dialog
- **当** 用户在镜像列表的操作菜单中点击"编辑"
- **那么** 系统必须弹出 Dialog（Modal）表单，预填充该镜像的当前配置，允许修改所有字段

#### 场景:添加镜像表单 - 映射端口输入
- **当** 用户在镜像表单中填写映射端口
- **那么** 页面必须展示单行文本输入框，placeholder 提示逗号分隔格式（如 `3000, 5173, 8080`），提交时作为字符串传递给 API

#### 场景:添加镜像表单 - 镜像选择
- **当** 用户在镜像表单中选择 Docker 镜像
- **那么** Docker 镜像字段必须展示为下拉选择组件，数据源为 `GET /api/docker/images` 返回的宿主机镜像列表，支持搜索过滤

#### 场景:添加镜像表单 - 环境变量编辑
- **当** 用户在镜像表单中配置环境变量
- **那么** 页面必须展示 key-value 表格组件，每行包含变量名输入框、值输入框和删除按钮，底部提供"添加变量"按钮。提交时前端必须将表格数据序列化为 JSON 字符串
