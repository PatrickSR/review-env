## ADDED Requirements

### 需求:测试容器持久化存储
系统必须将测试容器的元数据持久化到 SQLite `test_containers` 表中。表结构必须包含 `id`、`container_id`、`container_name`、`image`、`host_port`、`created_at`、`last_accessed_at` 字段。

#### 场景:创建测试容器时写入数据库
- **当** 通过 `POST /api/docker/test` 成功创建新的测试容器
- **那么** 系统必须将容器信息写入 `test_containers` 表，`created_at` 和 `last_accessed_at` 设为当前时间戳

#### 场景:停止测试容器时删除记录
- **当** 测试容器被停止（手动停止、超时清理或被新容器替换）
- **那么** 系统必须从 `test_containers` 表中删除对应记录

#### 场景:服务重启后恢复测试容器状态
- **当** Review Service 重启并执行 `recoverState()`
- **那么** 系统必须读取 `test_containers` 表中的记录，对每条记录检查 Docker 容器是否实际存在，删除已不存在的记录

### 需求:测试容器全局单例
系统必须保证全局最多只有 1 个测试容器在运行。

#### 场景:同镜像复用已有容器
- **当** 用户请求测试某个镜像，且已有一个使用相同镜像的测试容器在运行
- **那么** 系统必须直接返回已有容器的连接信息，并更新 `last_accessed_at` 为当前时间戳，禁止创建新容器

#### 场景:同镜像但容器已死亡
- **当** 用户请求测试某个镜像，DB 中有相同镜像的记录但 Docker 中容器已不存在
- **那么** 系统必须清理 DB 记录，创建新的测试容器

#### 场景:不同镜像替换旧容器
- **当** 用户请求测试某个镜像，且已有一个使用不同镜像的测试容器在运行
- **那么** 系统必须先停止并移除旧容器（包括 Docker 容器和 DB 记录），再创建新容器

#### 场景:无已有容器
- **当** 用户请求测试某个镜像，且 `test_containers` 表为空
- **那么** 系统必须创建新的测试容器

### 需求:测试容器超时清理
系统必须基于 `last_accessed_at` 字段定期清理超时的测试容器。超时时间通过 `TEST_CONTAINER_TIMEOUT_MINUTES` 环境变量配置，默认 30 分钟。

#### 场景:超时清理
- **当** `cleanupExpired()` 定期执行时
- **那么** 系统必须检查 `test_containers` 表中 `last_accessed_at` 距当前时间超过配置超时的记录，停止并移除对应的 Docker 容器，删除 DB 记录

#### 场景:活跃容器不被清理
- **当** 用户在超时时间内复用了测试容器（`last_accessed_at` 已刷新）
- **那么** 该容器禁止被超时清理
