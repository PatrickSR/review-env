## 目的

定义终端页面 React 组件，包含镜像选择、容器状态轮询与 IDE 显示、控制面板。

## Requirements

### 需求:终端页面 React 组件
系统必须提供 `/mr/:projectId/:mrIid` 路由对应的 React 页面组件，替代原有的独立 HTML 页面。该组件必须纳入 SPA 的 Layout 中，共享侧栏导航。

#### 场景:访问终端页面
- **当** 用户访问 `/mr/42/17`
- **那么** SPA 必须渲染终端页面组件，左侧显示侧栏导航，中间为终端主内容区

### 需求:MR 状态验证
终端页面加载时必须先验证 MR 的有效性。如果 MR 不存在、已关闭或已合并，必须显示友好的错误提示。

#### 场景:MR 有效（open 状态）
- **当** 终端页面加载且 MR 验证返回 valid=true
- **那么** 页面继续正常流程（查询容器状态或显示镜像选择）

#### 场景:MR 无效（已关闭/已合并/不存在）
- **当** 终端页面加载且 MR 验证返回 valid=false
- **那么** 页面必须在中间区域显示错误提示："该 MR 已关闭或不存在，请检查 MR 状态"
- **并且** 禁止显示镜像选择或终端 iframe

### 需求:镜像选择流程
当 MR 有效但无运行中的容器时，终端页面必须显示镜像选择界面，用户选择后启动容器。

#### 场景:无容器，显示镜像选择
- **当** MR 有效且容器状态为 not_found
- **那么** 页面必须在中间区域显示该项目可用的镜像列表（卡片形式）
- **并且** 用户点击镜像卡片后调用 `POST /mr/:projectId/:mrIid/start` 启动容器

#### 场景:无可用镜像
- **当** MR 有效但项目未配置任何启用的镜像
- **那么** 页面必须显示提示："该项目尚未配置任何镜像，请联系管理员"

### 需求:容器状态轮询与 IDE 显示
当容器存在时，终端页面必须轮询容器状态，就绪后显示控制面板并提供跳转到 code-server 的入口。

#### 场景:容器就绪
- **当** 容器状态轮询返回 status=ready 且 ports 包含 8080 映射
- **那么** 页面必须显示控制面板，包含"打开 IDE"按钮
- **并且** "打开 IDE"按钮点击后必须调用 `window.open` 在新标签页打开 `http://${location.hostname}:${ports["8080"]}/`

#### 场景:容器初始化中
- **当** 容器状态为 creating 或 initializing
- **那么** 页面必须显示加载动画和状态文字（"正在创建容器..."/"正在初始化环境..."）
- **并且** 每 2 秒轮询一次状态

#### 场景:容器出错
- **当** 容器状态为 error
- **那么** 页面必须显示错误信息

### 需求:控制面板
容器就绪后，终端页面必须显示控制面板，集中展示 MR 信息、IDE 入口、端口映射和容器操作。

#### 场景:控制面板内容
- **当** 容器状态为 ready
- **那么** 控制面板必须显示：MR 标题和分支信息、当前镜像名称、"打开 IDE"按钮、端口映射列表、"停止容器"按钮

#### 场景:端口映射显示
- **当** 控制面板显示端口映射
- **那么** 必须过滤掉 8080 端口（code-server 自身），仅显示用户配置的应用端口
- **并且** 每个端口映射必须可点击，点击后在新标签页打开 `http://${location.hostname}:${hostPort}`

#### 场景:停止容器
- **当** 用户点击"停止容器"按钮
- **那么** 必须弹出确认对话框，确认后调用停止 API 并将页面状态重置为 not_found
