## 目的

定义镜像级别的端口映射配置，包括数据模型扩展、容器创建时的端口映射、以及前端表单支持。

## Requirements

### 需求:镜像端口配置
每个镜像配置必须支持 `ports` 字段，用于定义容器创建时需要映射到宿主机的端口列表。格式为逗号分隔的端口号字符串。当 `ports` 为空字符串时，容器必须只映射内置的 ttyd 端口。

#### 场景:配置多个映射端口
- **当** 镜像配置的 ports 为 `"3000,5173,8080"`
- **那么** 创建容器时必须将容器内 3000、5173、8080 端口各自映射到 Docker 随机分配的宿主机端口

#### 场景:无额外端口配置
- **当** 镜像配置的 ports 为空字符串
- **那么** 创建容器时必须只映射内置的 ttyd 7681 端口

#### 场景:端口解析容错
- **当** ports 字符串包含空格或空项（如 `"3000, ,5173"` ）
- **那么** 系统必须忽略无效项，只映射有效的端口号

### 需求:ttyd 端口内置映射
ttyd 终端端口 7681 必须始终自动映射到宿主机随机端口，禁止依赖用户配置。该端口禁止出现在用户可见的端口配置界面中。

#### 场景:ttyd 端口始终映射
- **当** 创建任意容器（无论 ports 配置如何）
- **那么** 容器的 7681 端口必须映射到宿主机随机端口

### 需求:端口映射信息持久化
容器创建后，系统必须通过 `container.inspect()` 获取所有端口的实际宿主机映射，并将映射信息存入数据库的 `ports` JSON 字段。

#### 场景:记录实际端口映射
- **当** 容器创建成功且镜像配置了 ports `"3000,5173"`
- **那么** 数据库 containers 表的 ports 字段必须包含 7681、3000、5173 三个端口的实际宿主机映射

### 需求:端口配置数据模型
`project_images` 表必须包含 `ports` 列（TEXT 类型，默认空字符串）。API 的创建和更新接口必须支持 `ports` 字段的读写。

#### 场景:创建镜像时指定端口
- **当** 通过 `POST /api/projects/:id/images` 提交 `{ ..., ports: "3000,5173" }`
- **那么** 系统必须将 ports 保存到数据库

#### 场景:更新镜像端口配置
- **当** 通过 `PUT /api/projects/:id/images/:imageId` 提交 `{ ports: "8080,6006" }`
- **那么** 系统必须更新该镜像的 ports 字段

#### 场景:创建镜像时未指定端口
- **当** 通过 `POST /api/projects/:id/images` 提交时未包含 ports 字段
- **那么** ports 必须默认为空字符串
