## 目的

定义升级版侧栏导航，项目+MR 树形结构与全局管理导航。

## Requirements

### 需求:项目+MR 树形导航
侧栏上半部分必须按项目分组展示 MR 列表，形成树形导航结构。项目列表来自本地数据库，MR 列表来自 GitLab API（懒加载）。

#### 场景:侧栏初始加载
- **当** SPA 加载时
- **那么** 侧栏必须请求 `GET /api/projects` 和 `GET /api/containers` 获取项目列表和活跃容器
- **并且** 所有项目默认折叠显示
- **并且** 禁止在初始加载时调用 GitLab API

#### 场景:展开项目
- **当** 用户点击项目名称展开
- **那么** 侧栏必须调用 `GET /api/projects/:id/mrs` 获取该项目的 open MR 列表
- **并且** 首次展开时显示加载状态
- **并且** MR 列表加载完成后缓存结果

#### 场景:MR 列表显示
- **当** 项目展开且 MR 列表加载完成
- **那么** 必须显示所有 open 状态的 MR
- **并且** 有活跃容器的 MR 必须标记为活跃状态（🟢 图标）
- **并且** 活跃 MR 排在列表前面

#### 场景:收起再展开使用缓存
- **当** 用户收起项目后再次展开
- **那么** 必须使用缓存的 MR 列表，禁止重新请求 GitLab API

#### 场景:刷新按钮
- **当** 用户 hover 项目名称
- **那么** 必须在项目名称右侧显示刷新按钮（🔄 图标）
- **并且** 点击刷新按钮必须清除该项目的 MR 缓存并重新请求 GitLab API

#### 场景:点击 MR 导航
- **当** 用户点击某个 MR 项
- **那么** 必须导航到 `/mr/:gitlabProjectId/:mrIid`
- **并且** 如果该 MR 有活跃容器，终端页面直接进入终端显示
- **并且** 如果该 MR 无容器，终端页面显示镜像选择

### 需求:全局管理导航
侧栏下半部分必须保留现有的全局管理导航，与项目树之间有视觉分隔。

#### 场景:全局导航显示
- **当** 侧栏渲染完成
- **那么** 项目树下方必须有分隔线，分隔线下方显示全局导航
- **并且** 全局导航包含：首页、项目管理、镜像管理、容器监控
- **并且** 导航项的路由分别为 `/`、`/projects`、`/images`、`/containers`

#### 场景:导航高亮
- **当** 用户处于某个页面
- **那么** 侧栏中对应的导航项必须高亮显示
- **并且** 如果用户在终端页面，对应项目和 MR 项必须高亮
