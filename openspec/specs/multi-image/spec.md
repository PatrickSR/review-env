## 目的

定义多镜像配置、工具选择 UI 和容器切换（销毁重建）机制。

## Requirements

### 需求:项目镜像配置
每个项目必须支持配置多个 Docker 镜像，每个镜像代表一种 AI 工具环境。镜像配置必须包含名称、显示名、Docker 镜像地址、额外环境变量、映射端口和排序顺序。

#### 场景:添加镜像配置
- **当** 管理员通过 `POST /api/projects/:id/images` 提交镜像配置（name、display_name、image、env_vars、ports）
- **那么** 系统必须将镜像配置保存到数据库并关联到对应项目

#### 场景:镜像名称唯一性
- **当** 同一项目下添加重复 name 的镜像
- **那么** 系统必须返回 409 冲突错误

#### 场景:获取项目镜像列表
- **当** 请求 `GET /api/projects/:id/images`
- **那么** 系统必须返回该项目所有已启用的镜像配置（包含 ports 字段），按 sort_order 排序

#### 场景:禁用镜像
- **当** 管理员将某个镜像的 enabled 设为 false
- **那么** 该镜像禁止出现在终端页面的工具选择列表中

#### 场景:编辑镜像配置
- **当** 管理员通过 `PUT /api/projects/:id/images/:imageId` 提交更新字段
- **那么** 系统必须更新对应镜像的配置并返回更新后的完整记录

#### 场景:添加镜像表单 - 使用 Dialog
- **当** 用户在项目详情页点击"添加镜像"
- **那么** 系统必须弹出 Dialog（Modal）表单，包含标识名、显示名、Docker 镜像选择、映射端口和环境变量字段

#### 场景:编辑镜像表单 - 使用 Dialog
- **当** 用户在镜像列表的操作菜单中点击"编辑"
- **那么** 系统必须弹出 Dialog（Modal）表单，预填充该镜像的当前配置，允许修改所有字段

#### 场景:添加镜像表单 - 映射端口输入
- **当** 用户在镜像表单中填写映射端口
- **那么** 页面必须展示单行文本输入框，placeholder 提示逗号分隔格式（如 `3000, 5173, 8080`），提交时作为字符串传递给 API

#### 场景:添加镜像表单 - 镜像选择
- **当** 用户在镜像表单中选择 Docker 镜像
- **那么** Docker 镜像字段必须展示为下拉选择组件，数据源为 `GET /api/docker/images` 返回的宿主机镜像列表，支持搜索过滤

#### 场景:添加镜像表单 - 环境变量编辑
- **当** 用户在镜像表单中配置环境变量
- **那么** 页面必须展示 key-value 表格组件，每行包含变量名输入框、值输入框和删除按钮，底部提供"添加变量"按钮。提交时前端必须将表格数据序列化为 JSON 字符串

### 需求:终端页面工具选择
用户访问 `/mr/:projectId/:mrIid` 时，页面必须展示该项目可用的 AI 工具列表供用户选择。

#### 场景:展示工具选项
- **当** 用户打开 `/mr/42/17` 且项目 42 配置了 3 个已启用的镜像
- **那么** 页面必须展示 3 个工具选项卡供用户选择

#### 场景:项目无镜像配置
- **当** 用户打开 `/mr/42/17` 但项目 42 没有任何已启用的镜像配置
- **那么** 页面必须显示提示信息，告知管理员需要先配置镜像

#### 场景:选择工具并启动
- **当** 用户选择 "Claude Code" 并点击启动
- **那么** 页面必须发送 `POST /mr/42/17/start { imageId: 3 }` 创建容器

### 需求:基于镜像的容器创建
系统创建容器时必须根据指定的 imageId 从 project_images 表获取镜像名称和额外环境变量。

#### 场景:使用指定镜像创建容器
- **当** 收到 `POST /mr/42/17/start { imageId: 3 }` 且 imageId 3 对应镜像 `claude-code-fe:latest`
- **那么** 系统必须使用 `claude-code-fe:latest` 创建容器，并注入该镜像配置的额外环境变量

#### 场景:imageId 不属于该项目
- **当** 收到 `POST /mr/42/17/start { imageId: 99 }` 但 imageId 99 不属于项目 42
- **那么** 系统必须返回 400 错误

#### 场景:imageId 对应的镜像已禁用
- **当** 收到启动请求但指定的 imageId 对应的镜像 enabled=false
- **那么** 系统必须返回 400 错误

### 需求:容器切换（销毁重建）
同一项目同一 MR 同时只能运行一个容器。切换 AI 工具时必须先销毁旧容器再创建新容器。

#### 场景:切换工具
- **当** MR 17 已有一个 Claude Code 容器在运行，用户选择切换到 Codex
- **那么** 系统必须先停止并删除旧的 Claude Code 容器，再使用 Codex 镜像创建新容器

#### 场景:同一工具重新启动
- **当** MR 17 已有一个 Claude Code 容器在运行，用户再次选择 Claude Code 启动
- **那么** 系统必须销毁旧容器并创建新容器（全新环境）
